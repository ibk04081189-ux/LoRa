# LoRaWAN MAC Protocol Simulation

LoRaWAN ネットワークにおける 3 つの代表的な MAC プロトコルの衝突特性と PDR（Packet Delivery Rate: パケット配送率）を評価する Python シミュレーション・プロジェクトです。

## シミュレーション概要

920MHz 帯の LoRaWAN 通信環境を想定し、以下の 3 つのプロトコルを比較・評価します。

1.  **Pure ALOHA (LBTなし)**: 到着後、即座に送信を開始する基本モデル。
2.  **Pure ALOHA (LBTあり)**: 送信前に 5ms のキャリアセンス（LBT）を行い、空いている場合のみ送信するモデル。
3.  **Slotted ALOHA**: 送信時間を一定の「スロット」に限定し、衝突確率を抑制する同期モデル。

### 主な特徴
-   **2 勾配パスロスモデル**: 基準距離 D_BP を境に減衰定数が変化する現実的な伝搬環境を再現。
-   **固定シャドーイング行列**: 端末ペア間のチャネル状態をシミュレーション中一定に保つ高精度なピアセンシング（LBT）。
-   **SINR 評価**: 全イベント終了後に干渉電力を再計算し、未来のパケットによる干渉を漏れなく判定（Capture Effect 対応可能）。
-   **高速判定アルゴリズム**: `np.searchsorted` を活用し、500 台規模の端末数でも高速にシミュレーションが可能。

## ファイル構成

-   `sim_part2_completed_v2.py`: コンプリート版。全プロトコルの評価と CSV 出力を行います。
-   `sim_part2_hole.py`: 演習用（後輩向け）。主要ロジック（パスロス計算、ALOHA 衝突判定、LBT ロジック等）が `TODO` になっており、穴埋め課題として利用できます。
-   `results/`: シミュレーション結果の CSV ファイルがタイムスタンプ付きで保存されます。

## 使用方法

### セットアップ
```bash
# numpy 等の必要なライブラリをインストール（必要に応じて）
pip install numpy
```

### 実行
コンプリート版を実行して結果を取得します。
```bash
python3 sim_part2_completed_v2.py
```

実行後、`results/sim_part2_pdr_YYYYMMDD_HHMMSS.csv` が生成されます。

## 出力内容

CSV ファイルには以下の項目が出力されます。
-   `N`: 端末数（100〜500）
-   `Offered_Traffic_G`: 総トラフィック強度
-   `Pure ALOHA (LBTなし) (Sim)`: シミュレーションによる PDR
-   `Pure ALOHA Theory`: 純粋 ALOHA の理論値（e^-2G）
-   `Pure ALOHA (LBTあり) (Sim)`: LBT 導入時の PDR
-   `Slotted ALOHA (Sim)`: シミュレーションによる PDR
-   `Slotted ALOHA Theory`: スロット ALOHA の理論値（e^-G）

## 技術的詳細

### 1. 伝搬モデル (Propagation Model)
本シミュレーションでは、市街地環境を模した **2 勾配パスロスモデル** を採用しています。
-   **近距離 (d ≤ D_BP)**: 自由空間伝搬に近い減衰（2.0 勾配）。
-   **遠距離 (d > D_BP)**: 回折や遮蔽の影響を考慮した強い減衰（3.0 勾配）。
-   **シャドーイング**: 端末ペアごとに `sh_peer` 行列を事前生成し、LBT のキャリアセンス時と実際のデータ受信時で一貫したチャネル状態を再現しています。

### 2. MAC プロトコル実装
-   **Pure ALOHA (LBTなし)**: パケット発生と同時に送信。最も衝突が多い。
-   **Pure ALOHA (LBTあり) / CSMA**: 送信前に 5ms のキャリアセンスを行い、受信電力が `-80 dBm` (P_CS_W) 未満なら送信を開始します。ビジーな場合はランダムなバックオフ（0〜15スロット分）のあと再試行します。
-   **Slotted ALOHA**: `100ms` 固定のスロット境界でのみ送信。パケットが重なる場合はスロット全体が衝突対象となります。

### 3. 高速衝突判定アルゴリズム (SINR 一括計算)
シミュレーションの計算負荷を下げるため、以下のアルゴリズムを採用しています。
1.  全端末の送信イベント（開始・終了時刻）を `tx_log` に記録。
2.  全イベント終了後、開始時刻でソート。
3.  `np.searchsorted` を用いて、各パケットと時間的に重なる可能性のある前後パケットを $O(\log N)$ で抽出。
4.  抽出されたパケット群との SINR を一括計算。これにより、数千パケットの評価を数秒で完了させます。

### 4. LBT の 2 段階イベントモデル
LBT の 5ms 観測を再現するため、`heapq` を用いた離算事象シミュレーションを行っています。
-   **Event 0 (Arrival)**: 到着時点でのチャネル状態を確認。
-   **Event 1 (CS End)**: 5ms 後に再度確認。この両端でクリアな場合のみ「送信成功（開始）」とみなします。これにより、5ms の観測期間中に他者が送信を開始した場合の割り込みを疑似的に検知します。
